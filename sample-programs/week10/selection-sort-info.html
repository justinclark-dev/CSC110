    Selection Sort

    Selection Sort

    Sorting information is very common in computer software. We can sort numbers; we can sort strings based on length or alphabetically; we can choose to sort in ascending or descending order. Most modern programming languages, including Python, include built-in tools to sort a collection in a default manner.

    You may be surprised at the many ways there are to sort data a collection. One common sort algorithm is the Selection Sort; this algorithm works on any collection in any programming language. The task will be to sort numbers in ascending order. Once you understand sorting algorithms, you can adjust them to address any sorting need you may have.

    The Selection Sort

    The diagram below contains a flowchart and a table that explain how the Selection Sort works. It will help you immensely to slowly trace through the flowchart and to see that the numbers in the table are correct. Let's assume we have the following list:

    numbers = [3, 6, 1, 8, 4, 5]



    shaded part of list = the part that is searched (top to bottom)
    red number = smallest value in the shaded section
    arrows = swap

    The algorithm starts by setting idx to 0. It then searches through the entire list for the smallest value. It finds 1. It then swaps the value 1 with the value at index 0, which is 3. The result is shown in the column labeled After pass 1. Notice that the 1 and 3 have been swapped.

    The value of idx is incremented to 1. The algorithm then searches from subscript idx, which is the second element in the list, through to the end for the smallest value (this is shown in the column by the shaded area. Only the shaded area is searched). It finds 3. It then swaps the 3 with the second element (located at idx 1), which is a 6. The result is shown in the column labeled After pass 2. Notice at this point that the 1 and 3 have been put into their proper sorted positions.

    The table shows how the process completes. Here are a few features of the selection sort algorithm:

    • The algorithm is made up of passes. One pass consists of searching through the unsorted section of the list for the smallest value, and then swapping that smallest value into its correct location.
    • The idx variable represents 2 things: the beginning of the unsorted portion of the list to search, and where to place the smallest value once it has been found in the unsorted portion of the list.
    • The algorithm searches the list for the smallest value - nothing is changed during the search. The significant difference here from the search you've seen before is that it does NOT always start at index 0. If it did, we would always find the smallest value (the 1 in this example). We need to search for the smallest value in the portion of the list that is not already sorted. That's why we use idx as the starting index of the search.
    • The algorithm finds the index of the smallest value, not just the smallest value. As long as we have the index, we can always get access to that value by going into the list. The index is important when we do the swap.
    • only two values are swapped after the search finishes. The swap is based on the indexes of the values in the list.

    Here is the code for the selection sort.

    # selection_sort.py
    #
    # Selection Sort demo

    # CSC 110
    # Fall 2011

    def sort(numbers):

    # This outer loop keeps track of where the
    # unsorted portion of the list begins.
    for idx in range(len(numbers)-1):
    # Each time we come into the body of this outer loop, we are doing
    # one pass. Remember that a pass is made up of a search and a swap.
    #
    # Here is the code to search for and find the smallest value. We've
    # seen this before and know that it uses a for loop (so we've got
    # nested loops).
    # What we really need to know is the INDEX of the smallest value,
    # so we'll keep track of that. As long as we have an index, we can
    # go back into the list to get the value.
    #
    # Remember, we only want to search in the unsorted portion of the
    # list. Where does that section begin? At index idx.

    idxSmallest = idx; # initialization
    for i in range(idx + 1, len(numbers)):
    if numbers[i] < numbers[idxSmallest]:  # found a new smallest value,
    idxSmallest = i;  # so remember its subscript

    # now that we've found the index of the smallest value, swap it into place
    temp = numbers[idx]
    numbers[idx] = numbers[idxSmallest]
    numbers[idxSmallest] = temp
    # End of function, but nothing to "return" -- the list itself has been changed!


    # Main Program
    test = [3, 6, 1, 8, 4, 5] # fill a list with some numbers
    print('Original:', test)
    sort(test)
    print(' Sorted:', test)
