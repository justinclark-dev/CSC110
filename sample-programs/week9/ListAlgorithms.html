Problem Solving with Lists (and Sequences in General)

A list holds a collection of related data.  Typically, an indexed list of the kind we are examining would hold a collection of related values, each of which represents the same kind of thing.  For example, we might work with a set of test scores or a group of names.  If the data stored in the list are chosen carefully, then it is possible to perform many useful operations on the data. 

Finding the Largest Value

Imagine a list contains a series of interest rates offered by banks on savings accounts.  Write a program to find the highest interest rate.  Think about how you would solve this problem by hand.  Imagine that instead of a Python list, you have a stack of papers, each showing one interest rate.  How would you find out what the highest rate is?  

I'm sure you could do it even if you were allowed to look at each paper only once.  All you need to do is remember one number -- the highest rate you have seen.  If you see a rate that is higher, then remember the new number (and forget the previous one).  This one number you need to remember corresponds to a variable in your program -- an accumulator, in fact, since we will use it in a loop and it will accumulate information as the loop progresses.

The process of looking at each piece of paper one time corresponds to the loop you will use in the program to solve the problem.  We would say that you need to iterate through the list, examining it one element at a time.  Here is a program that implements this solution:

 # Find the highest value in 'nums'.
 # len(nums) assumed to be at least 1.
 def find_highest(nums):
     highest = nums[ 0 ]
     for ix in range(1, len(nums)):
         if nums[ ix ] > highest:
             highest = nums[ ix ]
     return highest

 interestRates = [1.5, 0.8, 3.5, 2.0, 1.8, 1.9]
 ans = find_highest(interestRates)
 print('Highest rate is ' + str(ans) + '%')
  # Test results: Printed 
  # in the shell window:
  
  Highest rate is 3.5%
  
 

Notice the following things about this example:

  • An accumulator called 'highest' is used in this loop.  This variable is initialized to the value of the first element in the list before the loop begins. 
  • Each subsequent element in the list is checked inside the loop.  We do this by causing the range function to produce a sequence that begins with 1.  The first element of the list is not checked inside the loop because it was handled during the initialization.
  • A decision is used inside the loop -- the value of 'highest' is changed only if the list element being checked is actually higher than the highest value already seen.
  • Since this process has been implemented inside a function, a return statement is used to send the highest value back to the calling statement.

Searching through a List

Searching is an extremely common task with lists.  The easiest way to find out if a particular value is present in a list is to perform a sequential search.  A sequential search involves iterating through the elements of a list in sequence and examining each element to see if it has the characteristics being sought.  Here is an example:

 # Returns True if the value 'target' is
 # in the list 'arr'; False otherwise.
 def find_in_list( arr, target ):
     for num in arr:
         if num == target:
             return True
     return False

 sizes = [6, 7, 8, 8.5, 9, 11]
 ans9 = find_in_list(sizes, 9);
 ans10 = find_in_list(sizes, 10);
 print('Have 9: '  + str(ans9) \
           + '; have 10: ' + str(ans10))
  # Test results: Printed 
  # in the shell window:
  
  Have 9: True; have 10: False
  

Notice the following things about this example:

  • The code to perform the search is written as a function.  Although this is not necessary (see next example), if the code is written this way, it can be re-used with other tests and/or other lists.
  • The goal of the search is only to determine if the target is present or not.  The function returns True if the target is present in the list, or False if it is not.
  • Both the list and the value being sought (the 'target') are passed into the function as parameters.
  • The loop iterates through all elements of the list.  An if statement is used to examine the value of each element.  If the target is found, the loop is terminated immediately and the function returns the value True.  If the entire loop completes without ever encountering the target value, the function returns False.
  • When the function is called, the order of the arguments in the function call must match the order of the parameters in the function definition -- the list must be provided first, followed by the target value.
  • This search function would work just as well with a list of strings. 
  • Sometimes it is more useful to know the index where the target was found, not just if it was found.  For example, the function might return the index where the target was found, or return a special value if the target was not found.  A common special value to return in other programming languages if the target is not found is -1 because -1 is not a legitimate index value in most programming languages, so it can act like a flag value.  In Python -1 is a legal index (pointing to the last element on the list), so one might return the special value None, or perhaps raise an exception (as the built-in 'index' method does).

Sifting (looking for multiple elements that match a certain criteria)

Sometimes, the goal of the search is to produce more than one result.  A search of this kind is often called a 'sifting' operation -- a name that calls to mind the process of sifting through a mixture to find elements of a particular size, weight, etc.  For example, we may wish to find all the values in a numeric list greater than a particular threshold, or maybe all the values in a string list beginning with a particular character.  Here is an example that finds all the elements that begin with "M":


  members = ['Homer', 'Marge', 'Lisa', 'Bart', 'Maggie']
  for ix in range(len(members)):
      if members[ix][0] == "M":
          print(members[ix], end=' ')
  
  # Test results: Printed 
  # in the shell window:
  
  Marge Maggie

Notice the following things about this example:

  • The loop must iterate through every element in the list, even if the target is found.  This is because it is possible for more than one element to meet the search criteria.
  • The search criteria are expressed as the condition of an if statement.  Any condition that can be tested in this way can be used as the basis for a search.  Imagine searching for all sizes between 10 and 12, or for all interest rates greater than or equal to 2%.
  • Some means must be provided to show multiple results.  In this example, the results are printed, so 'print( )' is called each time a matching element is found.  Another common approach is to create a new list and fill it with all elements in the original list that match the search criteria.  
  • A simple search or sift operation should not normally disturb (change) the list being searched.

Many variations on searches and sifts are possible.  In all cases, the basic principles are the same:

  • Iterate through all elements of the list.
  • Use an if statement to examine each element to see if it meets the search criteria.
  • Take appropriate action if a match is found.

The simple, or sequential, search described here works fine for small lists but can be inefficient for large lists.  Other, more efficient techniques (like binary search) are available, but are beyond the scope of this course.
